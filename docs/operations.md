# 運用ルール（初版）

## 目的
このサイトは **ER clinical tips の参照用ドキュメント**。内容は教育・実務補助であり、最終判断は担当医の臨床判断に基づく。

## URL運用
- **本番URL**: GitHub Pages（`main/master` pushで自動更新）
- **レビュー**: Pull Request上で内容確認してからマージ
- **更新単位**: 1トピック1PR推奨

## 更新フロー
1. 変更内容を `docs/*.md` に反映
2. PR作成（タイトルは `[tips] 変更概要`）
3. レビュー（医療内容・日本語表現・リンク切れ）
4. マージ
5. GitHub ActionsでPages自動反映
6. 更新履歴に追記

## 更新履歴テンプレ
- `YYYY-MM-DD`: `ページ名` を更新（要点: ...）

## 緊急改訂フロー
- 患者安全に関わる修正は優先で即時PR → 最小レビュー → 即時マージ
- マージ後、Discordで「緊急改訂」通知

## Discord通知テンプレ
```text
[ER Clinical Tips 更新]
- 変更: （例）胸痛ページに解離の赤旗を追記
- 影響範囲: chest-pain.md
- 公開URL: <ここにPages URL>
- 備考: 緊急改訂 / 通常更新
```
